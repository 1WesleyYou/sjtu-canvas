# reflection（auto-hw 子流程）

## 索引匹配

- **指定阅读材料** — 作业基础
- **作业要求和课程大纲** — 要求和课程介绍
- **先前阅读任务已经安排过的阅读材料** — 知识储备
- **先前完成的他人或自己的读后感** — 强化读后感要求，了解如何从阅读材料中提取写作角度，学习写作风格

不允许：
- 之后阅读任务的阅读材料
- 直接借鉴或抄袭之前的想法

## 流程

### 1. 写作

- **直接表达观点**，不堆背景信息
- **引用阅读材料的具体段落或论点**，不要泛泛说"作者认为……"
- **允许个人立场和质疑**——读后感的价值就在于学生自己的判断
- **长度**参考作业要求和所在目录里既有 reflection；没先例时默认 ~500 字左右
- **写作风格**参考之前作业内容

### 2. 落盘

- **先看作业所在目录里有没有既有 reflection 文件，有就沿用它的命名规则**
- 没先例 → `<阅读材料名字>_reflection.md`

### 3. Review

- 观点是否清楚明确
- 引用是否具体到位，不是泛泛复述
- 不堆背景、不空泛总结
- 删 AI 味（"本文将"、"综上所述"、"作为 AI"）
- 长度合理

## 风格

- Markdown 普通格式
- 第一人称口吻
- 不用 LaTeX 公式
